学情诊断算法文档

本文用尽量通俗的方式说明：系统如何从学生做过的题里，判断他可能掌握了什么、薄弱在哪里、为什么会错，以及下一题应该练什么。

1. 模型定位

当前 demo 的学情诊断是一个可解释原型，不是正式的心理测量模型。它的目标不是给学生下一个绝对结论，而是把作答记录转成老师和学生都能看懂的线索。

系统把一道题拆成三类诊断信息：

信息它回答的问题例子
知识点这道题主要考哪些课程内容？匀变速直线运动、静摩擦力、圆周运动
认知能力做这道题需要哪些跨题能力？图形表征、模型迁移、符号方向判断
错因如果做错，最可能是哪类错误？把实际静摩擦力误认为最大静摩擦力

这三类信息共同构成诊断结果。知识点和能力故意分开，因为“不会牛顿第二定律”和“不会从图里抽象模型”不是同一种问题。

2. 什么是 Q-matrix

Q-matrix 可以理解成一张“题目-知识点对照表”。它告诉系统：一道题涉及哪些知识点，以及每个知识点在这道题里有多重要。

在正式认知诊断模型中，Q-matrix 通常是一个 0/1 矩阵。例如一题需要“圆周运动”，就记 1；不需要“圆周运动”，就记 0。这个 demo 为了更贴近真实题目，使用了带权重的简化版本。

权重含义解释
1.0主要测量知识点这道题主要用来判断学生是否掌握它。
0.6辅助或前置知识点会影响解题，但不是这道题最核心的诊断目标。

例如第 1 题“拔河中的相互作用力与静摩擦力”对应的数据是：

"qMatrix": {
  "相互作用力": 1.0,
  "静摩擦力": 0.6,
  "受力分析": 0.6
}

这表示：如果学生做对第 1 题，系统会更明显地提高“相互作用力”的掌握判断，同时也略微提高“静摩擦力”和“受力分析”的判断。如果学生做错，则相反。

3. 知识点掌握如何更新

系统为每个知识点维护一个 0 到 1 之间的掌握概率。0 表示证据很弱，1 表示证据很强。新学生或没有证据的知识点，从 0.55 开始，相当于“暂时不确定，略偏中性”。

newScore = clamp(oldScore + direction * 0.14 * weight, 0.08, 0.96)
变量说明
oldScore该知识点原来的掌握概率。
direction答对为 1，答错为 -1。
0.14单题影响强度。它控制一题对诊断结果的影响不要过大。
weightQ-matrix 中该题对该知识点的权重。
clamp把结果限制在 0.08 到 0.96 之间，避免一两题就变成绝对判断。

用第 1 题举例，如果学生做对，主要知识点“相互作用力”的变化是：

0.55 + 1 * 0.14 * 1.0 = 0.69

辅助知识点“静摩擦力”的变化是：

0.55 + 1 * 0.14 * 0.6 = 0.634

这就是为什么同一道题会同时影响多个知识点，但影响程度不同。

4. 什么是能力载荷

能力载荷描述的是：一道题对某种跨题能力的依赖强度。它和知识点不同。

例如“匀变速直线运动”是知识点；“符号方向判断”是能力。学生可能知道公式，但一遇到正负方向就错，这时不能简单说他不会运动学，而应该指出他在符号方向处理上有薄弱。

当前 demo 中，每道题的能力载荷来自 agent/专家标注规则，而不是由大量学生数据训练出来。标注时主要看这些问题：

  1. 题目是否需要从图像、图表、物理情境中抽取关系？如果是，会给“图形表征”或“情境建模”较高载荷。
  2. 题目是否需要把已学模型迁移到新情境？如果是，会给“模型迁移”较高载荷。
  3. 题目是否容易因为方向、正负号、参考系而出错？如果是，会给“符号方向判断”较高载荷。
  4. 题目是否需要找临界状态、边界条件或分段过程？如果是，会给“临界思维”或“过程分段”较高载荷。

例如第 1 题的数据是：

"abilityLoadings": {
  "概念辨析": 0.85,
  "研究对象切换": 0.55
}

这表示：这道题主要在考学生能不能区分“绳对人”“人对绳”“地面对人”这些不同对象之间的力，所以“概念辨析”权重较高；同时它也要求在甲队、乙队、绳、地面之间切换研究对象，所以“研究对象切换”也有一定权重。

5. 能力证据如何汇总

能力不像知识点那样容易通过一两道题判断。当前 demo 采用“证据累计”的方式，而不是直接给出正式能力分。

if answer is correct:
  abilitySignal += loading
else:
  abilitySignal -= loading

abilityEvidence += loading

其中 abilitySignal 表示当前表现倾向，abilityEvidence 表示证据量。证据量太少时，页面会显示“证据不足”。

这样做的原因很简单：如果学生只做了一道图像题并且错了，我们不能马上断定他“图形表征能力弱”。只能说：当前有一个负向证据，还需要更多题确认。

6. 每题错因如何诊断

每道题都有一个或多个预设错因。错因不是随便写的标签，而是基于这道题最常见、最有教学意义的错误模式。

标注错因时，系统遵循三个原则：

  1. 错因要能解释学生为什么会选错，而不是只重复“这题不会”。
  2. 错因要能指导下一步练习，例如“比较对象错误”可以推荐同类比较题。
  3. 错因要尽量具体，例如“把实际静摩擦力误认为最大静摩擦力”比“摩擦力不会”更有用。

例如第 1 题保存的错因证据是：

"mistakeEvidence": {
  "把相互作用力大小误判为由质量决定": {
    "ifWrong": 0.85,
    "ifCorrect": 0.15,
    "feedback": "若本题出错，优先检查：把相互作用力大小误判为由质量决定。"
  }
}

意思是：如果学生做错这题，系统会认为这个错因很可能出现；如果做对，这个错因的风险会降低，但不会完全归零。

当前版本的错因诊断是“题目级”的：只要这题做错，就累积该题对应错因。更精细的下一版可以做到“选项级”：比如选 A 对应一种误区，选 B 对应另一种误区。那样诊断会更准确。

7. 推荐策略

推荐不是简单推“相似题”。系统会优先考虑学生为什么错。

  1. 如果某个错因出现，优先推荐同错因补救题。目标是纠正错误模式。
  2. 如果某个知识点掌握概率低，推荐同知识点锚定题。目标是确认基础是否稳定。
  3. 如果基础题表现较好，再推荐进阶迁移题。目标是检查能不能换情境应用。

在数据中，每道题有三类推荐关系：

"recommendations": {
  "similar": ["q002"],
  "remediation": ["q003"],
  "advanced": ["q004"]
}

remediation 用于补救，similar 用于巩固，advanced 用于提升。

8. 一个完整例子

假设学生做第 1 题，答案正确。系统会做几件事：

  1. 把“相互作用力”的掌握概率明显上调。
  2. 把“静摩擦力”和“受力分析”的掌握概率小幅上调。
  3. 给“概念辨析”和“研究对象切换”增加正向能力证据。
  4. 降低“把相互作用力大小误判为由质量决定”这个错因的风险。

如果学生做错，则方向相反：相关知识点掌握概率下降，相关能力出现负向证据，该题错因进入“主要错因”统计，并触发补练推荐。

9. 当前版本为什么不直接用 MIRT 或 G-DINA

MIRT 和 G-DINA 都需要足够多的学生作答数据来校准题目参数。当前 demo 只有 20 道题和本地单人作答记录，所以直接声称“正式估计能力”是不严谨的。

因此当前版本采用“专家标注 + 可解释更新”的方式：先把诊断逻辑跑通，让老师能看到题库数据结构、诊断解释和推荐闭环。等真实数据积累后，再升级统计模型。

10. 正式模型升级

后续可以按三步升级：

  1. 用真实作答数据验证 Q-matrix：检查某个知识点的题是否真的能区分掌握与未掌握。
  2. 用 MIRT 校准能力载荷：从大量学生作答模式中估计题目到底主要测哪些能力。
  3. 用 BKT 或纵向认知诊断跟踪变化：让系统知道学生练习后是否真的进步，而不是把所有历史记录平均到一起。

当前 demo 的意义，是先把这些高级模型需要的数据接口和产品体验准备好。
