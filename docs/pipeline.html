PDF 抽取 Pipeline 文档

本文说明如何把扫描版教辅 PDF 转成题库数据：先保留可核验的原题截图，再生成结构化题目、解析和诊断字段。

1. 输入与约束

源文件是扫描版 PDF，普通文本抽取几乎不可用。因此当前流程以页面渲染和题目区域裁切为主，而不是依赖 PDF 内置文本。

扫描版 PDF 的公式、图形、跨栏排版会显著增加自动切题难度。MVP 选择完整、单区域、可读的题目作为样本。

这个约束决定了 MVP 的优先级：先保证页面里看到的题目完整、可读、可回溯，再逐步提高自动 OCR 和结构化抽取能力。对于物理题来说，图像和公式比纯文字更关键，直接把题面截图作为第一等数据，可以避免早期 OCR 错误破坏演示体验。

PDF 情况处理方式原因
扫描图像页渲染为 PNG 后裁切题区。文本层不可用，必须从图像入手。
双栏排版优先选择单栏内完整题。跨栏题容易切断题干或选项。
图题和公式题保留原题截图。图像重画和公式重排成本高，且容易出错。
答案页分离解析生成后标记是否匹配答案页。不能假装全部自动校验完成。

2. Pipeline 步骤

  1. 用 pdftoppm 将 PDF 页面渲染为 PNG。
  2. 用人工或视觉 agent 标出完整题目区域。
  3. 用 Pillow 裁剪题目截图，保存到 assets/questions/。
  4. 将题号、来源页、题型、截图路径写入结构化数据。
  5. 对无法完全自动确认的内容添加质量标记。

这条流程中，“题目截图”和“结构化字段”是并行保留的。截图负责让学生看到原题，结构化字段负责让系统知道题目标题、答案、解析和诊断信息。

PDF -> 页面 PNG -> 题目截图 -> questions.js -> 页面练习与诊断

3. 当前脚本

scripts/build-crops.py 负责渲染和裁切。当前题目区域使用固定坐标，适合本 PDF 的 demo 样本。

python3 scripts/build-crops.py

脚本里维护了一个题目区域列表，每条记录包含题目 ID、PDF 页码和裁切坐标。例如：

("q001", 3, (40, 140, 480, 550))

含义是：第 1 题来自第 3 页，从页面图中裁切左上角 (40, 140) 到右下角 (480, 550) 的区域。这个方式不是最终自动化方案，但非常适合 MVP：可控、可复现、容易人工修正。

scripts/generate-question-data.py 负责生成 data/questions.js。它把题目标题、答案、解析、标签、诊断字段和推荐关系写成浏览器可以直接读取的数据。

python3 scripts/generate-question-data.py

scripts/validate-data.js 会检查 20 道题是否都有必要字段、截图文件、解析步骤和诊断标签。

4. 质量标记

标记含义
ocr_uncertain题干或公式来自扫描图，OCR/转写可能有误。
formula_needs_review公式需要人工核验。
answer_generated答案解析由 agent 推理生成，未完全匹配答案页。
answer_page_matched答案页中找到了可对应的答案或解析线索。
needs_teacher_review正式使用前需要老师核验。

质量标记不会展示给学生，但会保留在数据里。原因是：学生需要干净的练习体验，老师和开发者需要知道哪些题的内容尚未完全确认。

一个题目可以同时有多个质量标记。例如：

"qualityFlags": [
  "answer_generated",
  "ocr_uncertain",
  "formula_needs_review",
  "needs_teacher_review"
]

这表示题面来自截图，解析由系统生成，公式和答案需要后续人工核验。

5. 为什么不一开始就全自动 OCR

全自动 OCR 在纯文字文章里比较容易，但在高考物理教辅里会遇到三类问题：

  1. 公式识别错误，例如上下标、分式、根号和希腊字母容易错。
  2. 图像和题干关系复杂，OCR 只能读文字，不能天然理解图示中的力、速度、角度。
  3. 双栏和跨页排版会让题目边界识别困难，容易把上一题选项和下一题题干混在一起。

因此 MVP 采用“截图为准、结构化辅助”的路线。它不追求一次性自动转完整本书，而是先跑通从 PDF 到题库产品的闭环。

6. 后续自动化方向

后续可以用 OCR、版面检测、题号识别和图文区域合并替代固定坐标。对于跨页题，需要生成多图组合或转为结构化文本题干。

比较合理的升级路线是：

  1. 先用版面检测自动识别左右栏和题号位置。
  2. 再用 OCR 提取题干和选项文字，但仍保留原题截图。
  3. 用视觉模型判断图题中的关键物理对象，例如力、速度、角度、轨迹。
  4. 把答案页和题目页自动对齐，减少人工核验工作。
  5. 最后提供老师审核界面，允许编辑题干、答案、解析和标签。
