PDF 抽取 Pipeline 文档

本文描述如何从扫描版教辅 PDF 生成 demo 所需的题目截图和结构化草稿。

1. 输入与约束

源文件是扫描版 PDF，普通文本抽取几乎不可用。因此当前流程以页面渲染和题目区域裁切为主，而不是依赖 PDF 内置文本。

扫描版 PDF 的公式、图形、跨栏排版会显著增加自动切题难度。MVP 选择完整、单区域、可读的题目作为样本。

2. Pipeline 步骤

  1. 用 pdftoppm 将 PDF 页面渲染为 PNG。
  2. 用人工或视觉 agent 标出完整题目区域。
  3. 用 Pillow 裁剪题目截图，保存到 assets/questions/。
  4. 将题号、来源页、题型、截图路径写入结构化数据。
  5. 对无法完全自动确认的内容添加质量标记。

3. 当前脚本

scripts/build-crops.py 负责渲染和裁切。当前题目区域使用固定坐标，适合本 PDF 的 demo 样本。

python3 scripts/build-crops.py

4. 质量标记

标记含义
ocr_uncertain题干或公式来自扫描图，OCR/转写可能有误。
formula_needs_review公式需要人工核验。
answer_generated答案解析由 agent 推理生成，未完全匹配答案页。
answer_page_matched答案页中找到了可对应的答案或解析线索。
needs_teacher_review正式使用前需要老师核验。

5. 后续自动化方向

后续可以用 OCR、版面检测、题号识别和图文区域合并替代固定坐标。对于跨页题，需要生成多图组合或转为结构化文本题干。
